# Status: 2026-06-23

## Current Shape

The repo contains a v1 React + Node/TypeScript app for internal deck creation, Slidev preview, client sharing, and PPTX export/import.

Server code has been reorganized by domain under `apps/server/src/{agent,api,auth,core,db,decks,export,preview}`.

## Working

- Auth-gated app shell with first-admin bootstrap, magic-link login, invite flow, and admin user role/status management.
- SMTP is now reachable on `mail.21436587.xyz:465` with TLS and authentication.
- Admin/employee role boundary, deck ownership, collaborators, active editor locks, and share-link permissions.
- Scaffold/template curation from the Admin area.
- Deck creation, rough PPTX import, publish, markdown/PPTX export, export queueing, and stale export job reconciliation after restart.
- Draft preview builds are cached on disk with a `.slidev-agent-build.json`
  manifest and source hash. If cached output is stale, the server serves the
  previous cached preview while rebuilding in the background.
- Live Slidev preview supervisor with process reuse, crash retry/backoff, admin status panel, and stop control.
- Trial custom runtime scaffold at `themes/custom-html`, rendered from ordinary
  deck workspace files (`index.html`, `style.css`, `deck.js`) through the
  authenticated `/runtime/:deckId` route without Slidev, Vite, or a build step.
- Client share surface at `/client/:token`, with password and visitor identity gates.
- All agent edits now run through deepagents. It uses an OpenAI-compatible model
  provider underneath, but edits deck workspace files through path-jailed
  filesystem tools instead of returning plain replacement text.
- Structured request, export, and live-preview logs through pino.

## Recent UI Rule

Use Halfmoon components and utilities first. Custom CSS requires Ross's permission unless it is maintaining an already-approved fixed-format product surface such as preview frames or split workbench layout.

The sidebar has been moved to Halfmoon's sidebar/offcanvas pattern. Bootstrap JS
is now loaded for offcanvas drawer behavior and toast behavior, while Halfmoon
remains the visual component layer. The mobile sidebar drawer is full-screen.
Transient app notices now use toast markup instead of inline alerts.

## Known Gaps

- Dev login-link fallback should be put behind an explicit environment flag before any public production deployment.
- Deepagents smoke is opt-in and has not been run in this session with `RUN_DEEPAGENTS_SMOKE=true`.
- The UI still has some app-specific custom CSS for preview/workbench/dashboard surfaces; future UI work should reduce this where Halfmoon has a suitable component.
- Workbench live preview still starts a per-deck Slidev dev server; local edit
  mode now uses the direct localhost Slidev URL so Vite HMR is native instead of
  proxied through `/live/:deckId`. The first live startup can still be slow, but
  deck-detail draft previews should reuse the cached static build when source
  files have not changed.
- The custom runtime is still a trial path, but custom-html agent edits now
  require deepagents and target `index.html`, `style.css`, `deck.js`, and
  deck-local assets instead of `slides.md`.

## Preview Cache Notes

Static draft previews live under `.data/draft/:deckId`. The manifest stores the
source hash and build time. The cache validator checks for a real Slidev app
shell and relative asset references; it no longer relies on brittle generated JS
text such as an inline `routerMode` string.
